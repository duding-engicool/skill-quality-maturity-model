# 质量成熟度模型（QMM）维度与等级（内置知识）

> 通用成熟度分级思路，参考 ISO 9004 自我评定方法与 EFQM 卓越模型分级逻辑。权重与阈值企业可自定义（「待企业补充」）。

## 一、五级成熟度定义

| 等级 | 名称 | 典型特征 |
|------|------|----------|
| L1 | 初始级（应对式） | 质量问题靠个人救火，无稳定流程，结果波动大 |
| L2 | 已管理级（基本流程） | 关键过程有基本流程与职责，但执行一致性不足 |
| L3 | 已定义级（标准化） | 流程文件化、标准化，跨部门一致，有基本监控 |
| L4 | 量化管理级（数据驱动） | 过程绩效量化（KPI/SPC），数据驱动决策 |
| L5 | 优化级（持续改进/卓越） | 预防型改进、知识沉淀、对标卓越、持续创新 |

## 二、六评估维度

| 维度 | 关注内容 |
|------|----------|
| 1 领导与战略 | 质量方针、质量战略与组织战略衔接、资源承诺 |
| 2 顾客与满意 | 顾客声音（VOC）、满意度、投诉闭环、相关方 |
| 3 过程管理 | 过程识别、标准化、执行一致性、过程绩效监控 |
| 4 测量分析与知识 | 指标体系、趋势分析、知识管理、决策依据 |
| 5 改进与创新 | 纠正/预防措施、横向展开、预防型改进、创新 |
| 6 结果绩效 | 质量/成本/交付/效率结果，与目标的达成 |

## 三、维度等级特征（1–5 评分指引）

每个维度按以下特征打分：
- 5：系统化、数据驱动、持续对标卓越
- 4：量化管理，过程稳定可控
- 3：标准化、文件化，基本一致
- 2：有基本流程但执行不一致
- 1：临时应对，无稳定机制

## 四、整体等级映射（默认阈值，可调）

| 维度均分 | 整体等级 |
|----------|----------|
| ≥4.5 | L5 |
| ≥3.5 | L4 |
| ≥2.5 | L3 |
| ≥1.5 | L2 |
| <1.5 | L1 |

> 权重默认等权；企业可按战略对维度赋予不同权重（「待企业补充」）。
